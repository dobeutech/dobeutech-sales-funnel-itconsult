## 2024-04-08 - [CRITICAL] Client-Side Database Connections Exposed via VITE_ Prefix
**Vulnerability:** The Vite frontend (`my-neon-app`) directly instantiated a Neon database connection using `import.meta.env.VITE_DATABASE_URL`, exposing raw database credentials to the client bundle and allowing arbitrary remote code execution via direct SQL queries.
**Learning:** This existed because the frontend development environment was separate from the Next.js API layer, and developers bypassed the API layer by injecting the database URL directly into the Vite build step, unaware that `VITE_` variables are unconditionally bundled into public static files.
**Prevention:** Always maintain a strict "Backend-for-Frontend" (BFF) architecture. Never prefix database URLs or sensitive API keys with `VITE_` or `NEXT_PUBLIC_`. Implement pre-commit hooks to block commits containing sensitive `VITE_` variables. Ensure any client-side data fetching happens exclusively via authenticated Next.js API proxy routes.
