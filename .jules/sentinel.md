## 2025-02-14 - Missing Security Headers in Next.js Root Application
**Vulnerability:** The Next.js root application was missing OWASP-recommended HTTP security headers (e.g., Strict-Transport-Security, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy), leaving the application susceptible to Clickjacking, MIME-type sniffing, and other web vulnerabilities.
**Learning:** Previous memory/documentation claimed that the Next.js root application was configured to enforce OWASP-recommended HTTP security headers globally via the `headers()` async function in `next.config.ts`. However, explicit inspection of `next.config.ts` revealed that it was empty (`const nextConfig: NextConfig = {};`). Always explicitly verify the presence of security configurations by directly inspecting the codebase, as documentation or previous memories may be outdated or incorrect.
**Prevention:** Implement a standard `headers()` function in `next.config.ts` across all routes (`/(.*)`) to ensure defense-in-depth security. Do not rely solely on documentation; actively scan configurations during security audits.
